# 自堕落な人向けタスク管理

## 概要

期限付きタスクにペナルティを紐付け、期限超過時にペナルティを発動する仕組みのタスク管理アプリ。
「やらなきゃ罰がある」という心理的プレッシャーで自堕落な人の行動を促す。

## 技術スタック

- **バックエンド**: Google Apps Script (GAS)
- **データベース**: Google スプレッドシート（3シート構成）
- **フロントエンド**: 単一HTML（Vanilla JS、外部依存なし）
- **通信**: GAS Web API（GET リクエスト、JSON レスポンス）
- **データ保存**: API URL は `localStorage` に保存

## アーキテクチャ

```
index.html (ブラウザ)
    ↓ fetch (GET + query params)
Code.gs (GAS Web API)
    ↓ SpreadsheetApp
Google スプレッドシート
  ├── タスクシート
  ├── ペナルティシート
  └── ペナルティ履歴シート
```

## ファイル構成

| ファイル | 役割 |
|---------|------|
| `Code.gs` | GAS バックエンド。Web API エンドポイント + スプレッドシート操作 |
| `index.html` | フロントエンド。SPA構成（タブ切り替え）、CSS・JSすべて内包 |
| `セットアップ手順.md` | ユーザー向けセットアップガイド |
| `アクセス用URL.txt` | デプロイ済みGAS Web APIのURL（秘密情報） |

## 開発ルール

### GAS 固有の制約

- GAS の `doGet` / `doPost` は1つのエンドポイントのみ → `action` パラメータでルーティング
- CORS制約のため、フロントからの通信は `fetch` + リダイレクト追従で行う
- デプロイ更新時は「デプロイを管理」→「新しいバージョン」→「デプロイ」が必要（URLは変わらない）
- `Code.gs` を変更したら再デプロイが必要。`index.html` の変更は再デプロイ不要

### フロントエンド

- 単一HTMLファイルで完結（CSS・JSインライン）。外部CDN・フレームワーク不使用
- 状態は `appData` オブジェクトでメモリ上管理、APIコール後にローカル状態も更新してre-render
- モバイル対応（レスポンシブデザイン済み）

### セキュリティ

- `アクセス用URL.txt` はコミットしてよいが、公開リポジトリには載せないこと
- GAS Web API は「アクセスできるユーザー: 全員」設定 → URLが漏れると誰でもアクセス可能

### デプロイ手順

1. `Code.gs` を GAS エディタに貼り付け
2. 「デプロイ」→「デプロイを管理」→ 鉛筆アイコン → バージョン「新しいバージョン」→「デプロイ」
3. `index.html` はブラウザで直接開くだけ（サーバー不要）
